### Tool 1: search_listings

**What it does:**
Searches the mock dataset to find items that match the user's request, including the size and budget.

**Input parameters:**
- `description` (str): Search words the user provided, like "vintage tee" or "baggy jeans".
- `size` (str): The size they need, like "M".
- `max_price` (float): The maximum amount they are willing to spend.

**What it returns:**
A list of matching clothing items, sorted by relevance. Each item includes details like name, brand, price, and condition.

**What happens if it fails or returns nothing:**
If no items match the search, it returns an empty list. The main loop will see this, stop the search process, and ask the user to try different keywords.

---

### Tool 2: suggest_outfit

**What it does:**
Provides ideas on how to wear the newly found item by pairing it with clothing the user already owns.

**Input parameters:**
- `new_item` (dict): The specific item chosen from the search results.
- `wardrobe` (dict): The list of clothing items the user currently has in their closet.

**What it returns:**
A short styling suggestion explaining how to combine the new item with their existing clothes.

**What happens if it fails or returns nothing:**
If the user's wardrobe is empty, it will provide general fashion advice based on current trends instead of using owned items.

---

### Tool 3: create_fit_card

**What it does:**
Writes a short social media caption to show off the new purchase and the outfit idea.

**Input parameters:**
- `outfit` (str): The styling suggestion generated by the previous tool.
- `new_item` (dict): The new clothing item that was found.

**What it returns:**
A 1-3 sentence caption suitable for a post, mentioning the item, where it was found, and the overall style.

**What happens if it fails or returns nothing:**
If some details are missing, it uses a simple fallback message like, "Got a new piece! Can't wait to style it."

## Planning Loop

**How does your agent decide which tool to call next?**
The agent follows a set sequence. First, it runs `search_listings` using the user's search details. If it returns an empty list, the agent tells the user nothing was found and stops. If it finds items, it takes the first result and moves to `suggest_outfit` using the user's wardrobe. Once a styling idea is generated, it passes that idea and the new item to `create_fit_card`. Finally, it returns the fit card and the outfit advice to the user.

---

## State Management

**How does information from one tool get passed to the next?**
Information is kept in a session dictionary. This session tracks the `user_query`, the `search_results`, the `selected_item`, the `outfit_suggestion`, the `fit_card`, and any `error_message`. Each tool saves its output to this session, and the next tool reads what it needs from there.

## Error Handling

For each tool, describe the specific failure mode you're handling and what the agent does in response.

| Tool | Failure mode | Agent response |
|------|-------------|----------------|
| search_listings | No results match the query | "I couldn't find anything matching your search. Please try adjusting the size, price, or keywords." |
| suggest_outfit | Wardrobe is empty | "Your closet is empty right now, but you could easily style this piece with wide-leg jeans and chunky sneakers." |
| create_fit_card | Outfit input is missing or incomplete | "Got a new piece! Outfit details coming soon." |

## A Complete Interaction (Step by Step)

FitFindr searches for clothes using `search_listings`, figures out how to wear the best find using the user's closet with `suggest_outfit`, and writes a social media caption using `create_fit_card`. It does this in order. If it finds zero items, it stops and asks the user to try a different search.

**Example user query:** "I'm looking for a vintage graphic tee under $30. I mostly wear baggy jeans and chunky sneakers. What's out there and how would I style it?"

**Step 1:**
The agent calls `search_listings("vintage graphic tee", size="M", max_price=30.0)`. It returns 3 matches. It selects the top result: "Faded Band Tee — $22, Depop, Good condition."

**Step 2:**
The agent calls `suggest_outfit(new_item=<band tee>, wardrobe=<user's wardrobe>)`. It returns: "Pair this with your wide-leg jeans and platform Docs for a classic 90s grunge look. Roll the sleeves once and tuck the front corner slightly for shape."

**Step 3:**
The agent calls `create_fit_card(outfit=<suggestion string>, new_item=<band tee>)`. It returns: "thrifted this faded band tee off depop for $22 and honestly it was made for my wide-legs 🖤 full look in my stories"

**Final output to user:**
The user sees the selected item ("Faded Band Tee — $22, Depop"), the styling advice, and the social media caption. If the search had failed in Step 1, they would see: "I couldn't find anything matching your search. Please try adjusting the size, price, or keywords."
